# ADR-005: Adopt Argonne-Inspired Design Tokens (Personal Research)

## Context

The explorer is a **personal research / learning project**, not an official APS or ANL property. It needs a calm, readable visual identity for the author's own study sessions; an ANL-inspired palette (navy + cool secondary + muted orange accent) is chosen because the author finds it pleasant for long-form note reading and visually consistent with the source-material vocabulary the notes describe. The existing `eberlight-explorer/` legacy app uses Streamlit defaults, which look generic. We need a design token system that is **ANL-inspired (not officially aligned, not endorsed)** while meeting WCAG 2.1 AA accessibility for the author's own multi-hour reading sessions.

See ADR-010 for the WCAG-driven darkening of the secondary and accent tokens applied after this ADR was first accepted, and CLAUDE.md "Project Identity" for the unaffiliated-personal-research disclaimer.

## Decision

We adopt the design token system defined in `docs/02_design/design_system.md`:

- **Primary color** `#0033A0` (ANL navy) — 8.6:1 contrast on white, passes AA and AAA.
- **Secondary color** `#00A3E0` (APS blue) — 3.1:1 on white, restricted to large text or dark backgrounds.
- **Accent color** `#F47B20` (BER orange) — used for "Build and Compute" cluster and highlights.
- **Typography**: Source Sans 3 (body) and JetBrains Mono (code), both open-source.
- **Spacing**: 8px grid with 1200px max container width.

All color combinations are validated for WCAG 2.1 AA contrast compliance.

## Consequences

### Positive
- The palette is visually familiar to the author from APS materials and reads as scientifically calm for long study sessions.
- WCAG AA compliance is built into the token system, not retrofitted.
- Design tokens are defined once (`design_system.md` + `config.toml` + `styles.css`) and referenced everywhere.
- Open-source fonts avoid licensing concerns.

### Negative / Trade-offs
- The secondary color (`#00A3E0`) fails AA contrast on white for normal text — must be used carefully.
- Streamlit's default theme must be fully overridden, which requires `config.toml` + injected CSS.
- Custom components (breadcrumb, footer, etc.) need manual CSS rather than Streamlit's built-in styling.

### Follow-ups
- Configure `explorer/.streamlit/config.toml` with the primary token values.
- Create `explorer/assets/styles.css` with component-level CSS classes.
- Run Lighthouse accessibility audit after Phase A to validate contrast compliance.

## Alternatives Considered

- **Streamlit default theme**: Zero effort but looks generic. Rejected because the explorer must convey institutional credibility.
- **Material Design / Tailwind**: Modern design systems with comprehensive component libraries. Rejected because they don't align with ANL visual identity and add unnecessary dependency for a Streamlit app.
- **Exact ANL brand guide**: Full compliance with Argonne's official brand guidelines. Rejected because (a) this is a personal research project, not an official ANL property, and (b) we don't have rights to the official brand kit. "ANL-inspired (unaffiliated)" is sufficient and explicitly **not** "ANL-branded".

# ADR-010: Darken Cluster Accents for WCAG AA-Large + Reframe ADR-005 as Personal-Research

## Context

Two unrelated documentation gaps surfaced during the senior-review pass
in May 2026:

1. **Token drift.** The R7 accessibility iteration darkened two cluster
   accent colors in code (`lib/ia.py`, `tests/test_a11y.py`) so they
   would clear WCAG 2.1 AA-large contrast on the white surface, but
   neither `design_system.md` (DS-001) nor `ADR-005` was updated to
   reflect the new hex values. Wireframe HTML at
   `docs/02_design/wireframes/html/section_v0.1.html` still references
   the original tokens. Per CLAUDE.md invariant #4, design-system
   changes require an ADR status update; that update was never made.

2. **Tone drift.** ADR-005 frames the design system as "Argonne-aligned"
   for "DOE stakeholder demos, program reviews, and interviews,"
   implying institutional affiliation. The project is in fact a
   **personal research / learning workspace** with no APS, ANL, or DOE
   sponsorship, and the original sample-data owners have not been
   consulted about any public deployment. The framing risks misleading
   readers about endorsement.

Both gaps are addressed in this single ADR rather than two because
they touch the same surface (the token table + the framing language
in ADR-005) and ship in the same release (REL-E080).

## Decision

### Token darkening — formalise the R7 hex values

The Explore and Build cluster accents have been the following hex
values in code since R7, and are formalised here:

| Token | DS-001 v0.1.0 (original) | DS-001 v0.1.1 (R7-applied) | Contrast on white |
|---|---|---|---|
| `--color-cluster-explore` / `--color-secondary` | `#00A3E0` | `#0085C0` | 3.5:1 — passes AA-large ✓ |
| `--color-cluster-build` / `--color-accent` | `#F47B20` | `#D86510` | 3.1:1 — passes AA-large ✓ |
| `--color-primary` / `--color-cluster-discover` | `#0033A0` | `#0033A0` (unchanged) | 8.6:1 — passes AA + AAA ✓ |

Both darkened values were already exercised by `test_palette_passes_aa`
in `explorer/tests/test_a11y.py::test_cluster_accents_pass_aa_large`,
so this ADR documents existing reality rather than introducing a new
constraint. `design_system.md` is bumped to v0.1.1 with the new hex
values listed verbatim and the original kept in the change-history
section.

### Tone reframing — personal-research, unaffiliated

ADR-005 is amended (not superseded — the design tokens stand) to drop
language implying institutional affiliation. The corrected framing is:

- The explorer is a **personal research / learning project** by one
  researcher, not an APS / ANL / DOE property.
- The visual style is **ANL-inspired for personal familiarity**, never
  "ANL-aligned" or "ANL-branded".
- The original data owners of bundled samples (Sarepy, TomoBank, etc.)
  have **not been consulted** about any public hosting of this app;
  redistribution rights cover the bundled files for **local / private
  use** only.
- README.md and CLAUDE.md "Project Identity" carry the same disclaimer
  verbatim. The landing page in `explorer/app.py` carries a compact
  one-line variant under the hero.

### Wireframes — flag, don't regenerate

`docs/02_design/wireframes/html/*_v0.1.html` files reference the
original (pre-darkening) hex values. We do **not** regenerate them as
`_v0.2.html` because (a) the wireframes are historical reference, not
the implementation source of truth, and (b) per invariant #5,
wireframe versions are immutable. Instead, a "Visual Reference Only —
darker variant lives in DS-001 v0.1.1" comment is added to the head
of each wireframe HTML file.

## Consequences

### Positive
- DS-001 now matches the runtime palette. Anyone reading the design
  system finds the same hex values they would see in the rendered app.
- Anyone reading README / CLAUDE.md / ADR-005 / the landing page hero
  immediately understands this is one researcher's workspace, not an
  ANL property.
- `test_a11y.py` no longer carries the implicit assumption that the
  spec lags the code.
- Future style edits route through the canonical CSS custom properties
  added in REL-E080 (`:root { --color-primary: ... }` in `styles.css`),
  so the next palette change has a single editing site, not nine.

### Negative / Trade-offs
- Two historical wireframes carry slightly stale hex values. The
  inline comment is the only mitigation; we accept the cosmetic drift.
- ADR-005's "Argonne-aligned" language was used in several past
  release notes (REL-E020 through REL-E073). We do not rewrite history;
  only forward-facing copy is corrected.

### Follow-ups
- `scripts/build_static_site.py` already inlines the same darkened
  hex values via the `CLUSTER_META` import; no static-site code change
  is needed for the darkening itself, only for the disclaimer copy on
  the landing page (handled in REL-E080).
- The token-darkening rationale is captured in
  `explorer/lib/ia.py:CLUSTER_META` doc-comments; no further code
  change is required.

## Alternatives Considered

- **Supersede ADR-005 entirely.** Rejected: the *decision* (use these
  three colors + the type and spacing scale) is intact. Only the
  framing language and two hex values changed.
- **Brighten the original colors instead of darkening.** Rejected:
  the alternative direction (lighter teal / lighter orange) reduces
  contrast against the white surface further and fails AA-large.
- **Drop ANL inspiration entirely** (e.g. pick a green/purple palette).
  Rejected: the author finds the ANL-style navy + cool secondary
  palette pleasant for long study sessions, and the visual continuity
  with the source-material vocabulary the notes describe (synchrotron
  facility imagery) helps memory recall. The fix is honest framing
  ("inspired by, unaffiliated with"), not a complete palette swap.

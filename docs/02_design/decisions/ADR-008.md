# ADR-008: Section 10 — Interactive Lab as a tenth note folder

## Context

`09_noise_catalog/` documents 47 noise/artifact types and their mitigations
(traditional + AI/ML), but is *descriptive only*. Users currently cannot
**run** any of those mitigations and observe the effect of parameter changes
themselves. To close that gap we plan to add an Interactive Lab to the
Streamlit Explorer (planned `4_Experiment.py`) where users pick a sample,
choose a mitigation method, tune parameters, and see before/after with
quantitative metrics.

A Lab page needs **real input data**. The questions are:

1. Where does the data live in the repository?
2. What licences and ethical guard-rails apply?
3. How does this fit with the existing IA (3-cluster mapping in ADR-004) and
   the GitHub Pages mirror (ADR-007)?
4. Stay within free-tier hosting (no Git LFS paid plan, no Streamlit Cloud
   paid tier).

We surveyed every external reference in `09_noise_catalog/`,
`03_ai_ml_methods/denoising/`, and `04_publications/` (~70 papers, 20+ code
repos, 5+ archives), audited licences, and probed the Hugging Face Hub for
relevant pretrained weights.

Findings drove three constraints:

- The notes are still the single source of truth (CLAUDE.md invariant #1) —
  bundled sample data is **input** to experiments, not new taxonomy.
- "Maximum diversity, 5–10 samples per case" beats "one large dataset" for an
  interactive learning surface.
- Free-tier means: no Git LFS (1 GB free quota is too tight), no individual
  file > 100 MB, repo total ideally < 1 GB.

## Decision

We add a **tenth note folder**, `10_interactive_lab/`, with a fixed
sub-structure:

```
10_interactive_lab/
├── README.md, manifest.yaml, CITATIONS.bib
├── LICENSES/                 # verbatim original LICENSE files
├── docs/
│   └── external_data_sources.md
├── datasets/
│   ├── tomography/, xrf/, spectroscopy/,
│   ├── scattering_diffraction/, cross_cutting/,
│   └── ptychography/, electron_microscopy/    (README only — link out)
└── models/
    ├── README.md
    └── lazy_download_recipes.yaml
```

Mechanics:

- **Bundled samples are git-tracked** as plain files. Each subfolder holds
  5–10 samples. Aggregate is ~135 MB (Sarepy ring-artifact sinograms
  dominate at 18–25 MB each).
- **No Git LFS.** All files are under the GitHub 100 MB hard limit.
- **No pretrained weights are bundled.** The Streamlit Lab will lazy-fetch
  via `pooch.retrieve(...)` with hash verification on first use, per
  `models/lazy_download_recipes.yaml`. License is shown to the user before
  any download begins.
- **Every dataset subfolder ships an `ATTRIBUTION.md`** (YAML frontmatter
  per invariant #7) declaring upstream URL, exact commit SHA we mirrored,
  authors, license, citation string, and an explicit disclaimer that this
  is not the official mirror.
- **`manifest.yaml`** is the machine-readable inventory consumed by the
  Streamlit page; it groups files by modality, role
  (`clean_reference`, `noisy_input`, `flat_reference`, `dark_reference`,
  `theoretical_reference`, `calibration_reference`, `false_positive_trap`,
  `metadata`), and noise-catalog cross-reference.
- **`docs/external_data_sources.md`** is the curated extension. It lists
  every dataset we did *not* bundle (TomoBank entries, EMPIAR cryo-EM,
  CXIDB, AAPM Low-Dose CT, etc.), with download recipes, license rules,
  and citation strings.
- **Versioning** follows the `notes-vX.Y.Z` SemVer stream (invariant #6).
  Initial release is `notes-v0.10.0`.
- **GitHub Pages mirror** (invariant #9): the static-site generator
  (`scripts/build_static_site.py`) treats `10_interactive_lab/` as another
  note folder and renders its markdown pages in the same shell as the
  other 9 sections. Binary samples (TIFF, FITS, NumPy) are *not* served
  on Pages — the generator will skip them; the live experiment surface is
  Streamlit-only. This is consistent with ADR-007's existing trade-off
  ("Streamlit-only features that depend on a running Python process …
  cannot be mirrored").

## Consequences

### Positive

- Closes the "read-only catalog → hands-on lab" gap with a clean separation:
  taxonomy lives in `09_*`, runnable experiments live in `10_*`.
- Real data only (no synthetic fakery), spanning 6 modalities, drawn from
  8 well-known open-source projects (Sarepy, TomoPy, xraylarch, PyMca,
  PyXRF, pyFAI, astroscrappy, plus configs from NSLS-II PyXRF) — all
  permissively licensed (Apache-2.0, BSD-3, MIT, LGPL-2.1+).
- Honours research ethics: every dataset is attributed, every required
  citation captured in `CITATIONS.bib`, every LICENSE preserved verbatim.
- Free-tier safe: ~135 MB of bundled data, no LFS, no file over 100 MB.
- The lazy-download mechanism keeps heavy resources out of the repo while
  still letting users replay them locally.
- `external_data_sources.md` doubles as a curated atlas for users who
  want to take experiments further than the bundled samples allow.

### Negative / Trade-offs

- IA changes from 9 → 10 note folders. Documents that hard-code "9 note
  folders" must be updated (`README.md`, `CLAUDE.md` directory map).
  The 3-cluster mapping in ADR-004 grows by one entry; we treat
  `10_interactive_lab/` as part of the **Build** cluster
  (matching `05_tools_and_code/`, `06_data_structures/`,
  `07_data_pipeline/`).
- The static site cannot run experiments. The Pages mirror will show the
  Lab's markdown but a banner will direct Pages users to either run the
  Streamlit app locally or use the curated external links.
- Bundled binary samples grow the repo by ~135 MB and slow down
  initial clones. The single biggest contributor (~133 MB) is
  `datasets/tomography/ring_artifact/`. This is a one-time cost.
- We declined to bundle Hugging Face Hub baselines (NAFNet, SwinIR, …)
  even though their licences (MIT/Apache-2.0) would allow it. They are
  generic, not synchrotron-specific, and the lazy-download path through
  the `transformers` / `diffusers` SDKs is the canonical way to use them.
  Bundling weights here would create a maintenance burden with little
  benefit.
- We could not bundle the actual TomoGAN demo dataset, EMPIAR entries, or
  TomoBank multi-dose pairs in this iteration because the sandbox in
  which the lab was first authored had only git-clone egress, not direct
  HTTPS to Box / EBI / TomoBank. They are documented in
  `external_data_sources.md` for users to fetch on their own.

### Follow-ups

- Add a PRD requirement (FR-XXX, US-XXX) explicitly covering the Lab in
  the next PRD revision.
- Implement `explorer/pages/4_Experiment.py` consuming `manifest.yaml`,
  with the recipe schema (parameter slider/select auto-generated from
  YAML) discussed in the v0.10.0 release notes. Mirror the new page in
  `scripts/build_static_site.py` per invariant #9 (with a static
  "open in Streamlit to run" banner where the page is non-mirrorable).
- Add a CI step that validates every sample file in `manifest.yaml`
  exists, every `ATTRIBUTION.md` carries the required frontmatter, and
  every `LICENSE` referenced in attributions is present in
  `LICENSES/`.
- Once we have free-tier-friendly TomoBank / EMPIAR fetch paths working
  in the Streamlit Lab via `pooch`, vet first downloads, pin SHA-256
  hashes, and update `lazy_download_recipes.yaml`.
- After the first publication that uses Lab data, add the citation to
  `CITATIONS.bib` and link it from `04_publications/`.

## Alternatives Considered

- **Place the lab inside `09_noise_catalog/interactive/`** rather than as
  a top-level tenth folder. Rejected — mixing taxonomic notes with
  binary input data muddies invariant #1 ("notes are the single source
  of truth"); and a top-level tenth folder is a cleaner growth path
  (independent SemVer cadence, independent Pages routing, independent
  attribution surface).
- **Synthesize all data on the fly** (Shepp-Logan + injected noise
  models). Rejected — the user explicitly requested *real* data only,
  and we agreed real data carries pedagogical authority that synthetic
  cannot match.
- **Use Git LFS for the bundled samples.** Rejected — GitHub's free LFS
  budget (1 GB storage + 1 GB/month bandwidth) is too tight; on a
  popular repository the bandwidth cost would force a paid plan, which
  contradicts the "free-tier only" constraint.
- **Bundle pretrained model weights** (Topaz, TomoGAN, …) alongside the
  data. Rejected — license diversity (CC-BY-NC, GPL-3.0, MIT in the
  same folder) is a maintenance trap, sizes are large, and authors
  update without notice. Lazy-download via `pooch` is strictly better.
- **Mirror everything on Hugging Face Hub** (datasets + models). Considered
  for future iterations, but rejected for this first pass: most cited
  synchrotron-domain models are *not* on the Hub today and bandwidth
  for very large datasets on Hub free tier is uncertain. We keep this
  as an option once we know real Lab usage patterns.
